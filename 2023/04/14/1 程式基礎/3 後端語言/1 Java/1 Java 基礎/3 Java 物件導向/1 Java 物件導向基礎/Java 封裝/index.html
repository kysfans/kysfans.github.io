Java 物件導向基礎

Java 封裝（Encapsulation）是一個物件導向程式設計的概念，它指的是將類別的資料成員（屬性）和方法封裝在類別內部，並通過公共介面（公開方法）提供對這些成員的訪問。封裝的目的是隱藏類別的內部實現細節，只公開必要的方法來與外部進行交互，從而確保程式碼的安全性、可讀性和可維護性。

封裝具有以下優點：

  1. 資料隱藏：封裝可以將類別的內部資料隱藏起來，避免外部直接訪問和修改這些資料。這樣可以防止資料被意外改變，並確保類別的內部實現細節不被外部知道。

  2. 程式碼安全性：封裝可以減少程式碼中的錯誤，防止不正確或無效的資料訪問，從而提高程式碼的安全性和穩定性。

  3. 接口簡化：封裝使外部只需關心公開的介面（公共方法），而不需要了解實際的內部實現。這使得類別的使用更加簡單且易於理解。

  4. 程式碼隔離：封裝可以將類別的功能和實現細節隔離，從而降低類別之間的耦合度。這使得程式碼更容易修改和擴展。

在 Java 中，實現封裝的主要手段是使用訪問修飾符來控制類別的成員的可見性：

  • public：公開訪問修飾符，表示該成員可以在任何地方訪問。
  • private：私有訪問修飾符，表示該成員只能在類別內部訪問，外部無法直接訪問。
  • protected：受保護訪問修飾符，類似於 private，但允許子類別訪問。

例如，以下是一個使用封裝的範例： 

public class Person {  
    private String name;  
    private int age;  
  
    // 建構子  
    public Person(String name, int age) {  
        this.name = name;  
        this.age = age;  
    }  
  
    // Getter 和 Setter 方法  
    public String getName() {  
        return name;  
    }  
  
    public void setName(String name) {  
        this.name = name;  
    }  
  
    public int getAge() {  
        return age;  
    }  
  
    public void setAge(int age) {  
        this.age = age;  
    }  
}

在這個範例中，name 和 age 成員變數是私有的，通過公開的 Getter 和 Setter 方法來訪問和設置這些成員。這樣，外部無法直接訪問 name 和 age，只能透過方法來間接操作這些資料。

通過適當使用封裝，您可以保護程式碼的安全性，使類別更易於使用和維護，並提高程式碼的可靠性和可讀性。
